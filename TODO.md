## 1. Прогнать обновление данных локально (блокер для п.2)

```bash
node scripts/update-objectives.mjs
git add tasks.json && git commit -m "Refresh quest objectives from tarkov.dev" && git push
```

Из облачной песочницы Claude `api.tarkov.dev` недоступен (сетевая политика
окружения), поэтому скрипт нужно запустить с локальной машины. Что он дольёт
в `tasks.json`:

- цели-таргеты для всех ~510 квестов (сейчас только 209 — каппа-квесты
  из npm-пакета `tarkovdata`);
- **русские тексты целей** (`ru`-поле — приложение уже рендерит его при
  RU-интерфейсе автоматически);
- предметы целей и требуемые ключи **с иконками** (`iconLink`),
  полными названиями и короткими подписями (`shortName`) — данные под п.2.

## 2. Фича «Сборы в рейд» (предметы и ключи) — после п.1

Агрегированный чек-лист перед рейдом: выбираешь карту → по всем доступным
квестам этой карты собирается список **ключей** и **предметов на сдачу/поиск**
(с суммарными количествами и пометкой FiR).

Требования к UI (зафиксировано):

- **обязательно с иконками** предметов/ключей (`iconLink` из tarkov.dev,
  хотлинк на assets.tarkov.dev);
- **подписи** под/рядом с иконкой — короткое имя (`shortName`);
- **альты/тултипы при наведении** — полное название предмета (атрибуты
  `title` + `alt` на изображении), в идеале с указанием, каким квестам
  предмет нужен («2× Salewa (FiR) — Postman Pat, Shortage»).

Данные для этого появятся в `tasks.json` после прогона скрипта из п.1
(поля `objectives[].items[]` и `keys[]` у квестов). Реализация — агрегатор
поверх этих полей во вкладке «Рейды», ~150 строк, без новых зависимостей.
